Time in Use — Top 5 Models ⓘ
What this measures
For each model: % of business-hour 5-min windows (Mon–Fri 09:00–18:00 SGT) where ≥1 real user request was in flight. Top 5 ranked by this %. Fleet avg = card-weighted mean across all models.
Raw metric
litellm_requests_metric_total
PromQL (ETL input)
sum by (model) (increase(\n litellm_requests_metric_total{\n model!="unknown",\n team_alias!="litellm-internal-health-check"\n }[1m]))
Business-hours filter applied in ETL (Python), not in PromQL.

Utilization by Workload ⓘ
What this measures
Fraction of GPU compute units actively executing — how hard the cards are actually working. Measured 24×7 including weekends and overnight. Two clusters pulled separately and card-weighted (216 B200 : 20 H100) into a fleet %. Not the same as "Time in Use" (which counts request presence, not compute intensity).
Raw metric
DCGM_FI_DEV_GPU_UTIL
PromQL (per cluster)
avg( avg by (UUID) (\n DCGM_FI_DEV_GPU_UTIL{\n modelName="NVIDIA B200"\n }))
Same query with modelName="NVIDIA H100 NVL" for the H100 cluster. avg by (UUID) collapses MIG slices to physical card.

Token Output by Model ⓘ
What this measures
Daily output tokens generated per production model (tracked generative LLMs only — excludes embeddings, rerankers, OCR, CI/benchmark traffic). Stacked total = fleet throughput. Each day = one trailing-24h total.
Raw metric
litellm_output_tokens_metric_total
PromQL (ETL input)
sum by (model) (increase(\n litellm_output_tokens_metric_total{\n model!="unknown",\n team_alias!="litellm-internal-health-check"\n }[1d])) / 1e6
ETL takes the last reading per (model, day) — not a sum of 5-min rows.

Output tokens by model

Output-token generation per served model; the stacked total is fleet throughput, with peaks shifting as demand moves between models. A model-level lens on the same throughput shown by organisation below.

What this measures
Inference: output tokens and internal cost broken down by Organisation → Application → Model. Cost = tokens × admin-set $/1M governance rate. Training: GPU-hours and cost by org (not yet captured — Slurm exporter pending).
Raw metric (inference)
litellm_output_tokens_metric_total
with team_alias label → mapped to org via dim_applications.

sum by (org_alias, team_alias, model) (\n increase(\n litellm_output_tokens_metric_total{\n model!="unknown",\n team_alias!="litellm-internal-health-check"\n }[1d])) / 1e6

Cost per Million Tokens vs Open-Model Cloud Host ⓘ
What this measures
Output cost per 1M tokens: self-hosted internal rate vs the same model hosted externally. Shows whether self-hosting is cheaper. * = $0.20 placeholder (real governance rate not yet set).
Internal cost source
Admin-provided governance rate in dim_models.internal_cost_per_m_usd. Not metric-derived — set by the platform team.
Cloud price source
Manually curated from provider websites / APIs. Stored in fact_model_pricing.csv. No PromQL — admin-maintained input.
